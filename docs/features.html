Features & Tools

Discover all the built-in features and tools available in the SKU Specification Comparator application.

Core Features

AI-Powered Specification Generation

The application leverages Google Gemini 2.5 Flash to generate product specifications based on SKU identifiers.

  • Model: Gemini 2.5 Flash - optimized for speed and accuracy
  • Context-aware: Uses SKU information to provide relevant specifications
  • Structured output: Ensures responses match your exact attribute schema
  • Consistent format: Always returns data in the same structure as your input

Flexible Input Parsing

The application intelligently parses multiple input formats without requiring you to specify which format you're using.

JSON Input

{
  "Attribute1": "Value1",
  "Attribute2": "Value2"
}
  • Standard JSON object format
  • All values converted to strings automatically
  • Handles nested values by stringifying them

Colon-Separated Input

Attribute1: Value1
Attribute2: Value2
  • Simple line-by-line format
  • Automatic whitespace trimming
  • Supports colons within values (everything after first colon)

Visual Difference Highlighting

The application uses color-coded highlighting to make mismatches immediately obvious.

  • Subtle row highlighting: Entire rows with mismatches have an amber tint
  • Cell highlighting: LLM Value cells with differences are prominently highlighted
  • Color scheme: Amber/orange tones for warnings without being alarming
  • Accessible: High contrast ratios for visibility

Interactive Difference Filter

Toggle between viewing all attributes or only those with differences.

  • Quick toggle: Single click to switch views
  • Instant update: Table filters immediately without page reload
  • Contextual message: Shows count when all differences are hidden
  • Preserved state: Setting remembered across comparisons

Real-Time Comparison Summary

See at-a-glance statistics about your comparison:

  • Total attributes: Count of all compared specifications
  • Mismatch count: Number of differences detected
  • Perfect match indicator: Special message when all values align
  • Dynamic updates: Updates with each new comparison

Data Processing Features

Smart Comparison Logic

The comparison algorithm includes intelligent matching:

  • Case-insensitive: "Red" matches "red"
  • Whitespace trimming: Leading/trailing spaces ignored
  • Normalization: Values converted to lowercase for comparison
  • Exact display: Original values shown without modification

Structured Output Enforcement

Ensures AI responses match your schema exactly:

  • JSON schema validation: Response must match defined structure
  • Required fields: All your attribute keys must be present
  • No extra fields: AI cannot add attributes you didn't specify
  • Type safety: All values returned as strings

Automatic Format Detection

The parser tries JSON first, then falls back to colon-separated:

  • JSON parsing: Validates and extracts object properties
  • Fallback parsing: Splits on colons and line breaks
  • Error reporting: Specific feedback on which line failed to parse
  • Flexible handling: Converts non-string values to strings
