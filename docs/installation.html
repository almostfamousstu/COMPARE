Installation Guide

This guide walks you through installing and setting up the SKU Specification Comparator on your local machine.

Prerequisites

Before installing the application, ensure you have the following:

Requirement Version Purpose
Node.js 18.17.0 or newer JavaScript runtime for running the application
npm Included with Node.js Package manager for installing dependencies
Git Latest version Version control (if cloning from repository)
Google Gemini API Key N/A Required for AI-powered comparisons

Step 1: Check Node.js Version

The application requires Node.js version 18.17.0 or newer. Verify your Node.js version:

Terminal
node --version

If your version is older, download and install the latest LTS version from nodejs.org.

Using nvm? If you use nvm (Node Version Manager), the repository includes a .nvmrc file. Simply run: 
nvm use

Step 2: Clone or Download the Repository

If you have access to the source code repository, clone it:

Terminal
git clone <repository-url>
cd COMPARE

Alternatively, if you received the source code as a package, extract it to your desired location.

Step 3: Install Dependencies

Navigate to the project directory and install all required dependencies:

Terminal
npm install

This command will download and install all dependencies listed in package.json, including:

  • Next.js 14 - React framework
  • React 18 - UI library
  • @google/generative-ai - Google Gemini SDK
  • Tailwind CSS - Styling framework
  • shadcn/ui components - UI component library
  • TypeScript - Type checking

Step 4: Configure Environment Variables

The application requires a Google Gemini API key to function. Follow these steps to configure it:

4.1 Create Environment File

Copy the example environment file:

Terminal
cp .env.local.example .env.local

4.2 Obtain Google Gemini API Key

  1. Visit Google AI Studio
  2. Sign in with your Google account
  3. Click "Get API Key" or "Create API Key"
  4. Copy the generated API key
Important: Keep your API key secure. Never commit it to version control or share it publicly.

4.3 Update Environment File

Open .env.local in your text editor and add your API key:

.env.local
GEMINI_API_KEY="your-actual-api-key-here"

Step 5: Verify Installation

Start the development server to verify everything is working:

Terminal
npm run dev

You should see output similar to:

   ▲ Next.js 14.2.3
   - Local:        http://localhost:3000
   - Network:      http://192.168.1.x:3000

 ✓ Ready in 2.3s

Open your web browser and navigate to http://localhost:3000. You should see the SKU Comparator interface.

Troubleshooting Installation

Node.js Version Issues

If you encounter errors related to Node.js version:

  • Ensure you're using Node.js 18.17.0 or newer
  • If using nvm, run nvm use in the project directory
  • Consider using nvm to manage multiple Node.js versions

Installation Failures

If npm install fails:

  • Try clearing npm cache: npm cache clean --force
  • Delete node_modules and package-lock.json, then run npm install again
  • Check your internet connection
  • Ensure you have sufficient disk space

API Key Issues

If you see errors about missing API key:

  • Verify .env.local exists in the project root
  • Ensure the API key is properly formatted with quotes
  • Restart the development server after updating .env.local
  • Check that your API key is valid in Google AI Studio
