Frequently Asked Questions

Common questions and answers about the SKU Specification Comparator application.

General Questions

What is the SKU Specification Comparator?

The SKU Specification Comparator is a web application that helps you validate product specifications by comparing your data against AI-generated specifications from Google Gemini. It highlights discrepancies to help ensure data accuracy.

Who should use this application?

This tool is useful for:

  • Quality assurance teams validating product catalogs
  • E-commerce managers maintaining accurate product information
  • Data analysts checking specification consistency
  • Product managers verifying attribute accuracy
  • Anyone responsible for product data quality

Is this application free to use?

The application itself is free and open-source. However, you need a Google Gemini API key, which may have associated costs based on your usage. Google typically offers free tier access for development and testing.

Do I need to install anything?

Yes, you need:

  • Node.js 18.17.0 or newer
  • A Google Gemini API key
  • The application source code

Alternatively, you can run it with Docker if you prefer containerized deployment.

Setup & Configuration

How do I get a Google Gemini API key?

  1. Visit Google AI Studio
  2. Sign in with your Google account
  3. Click "Get API Key" or "Create API Key"
  4. Copy the generated key
  5. Add it to your .env.local file

Is my API key secure?

Yes. The API key is stored server-side only and never exposed to the browser. The .env.local file is included in .gitignore to prevent accidental commits to version control.

Can I use this without an API key?

No. The application requires a valid Google Gemini API key to function. The comparison feature relies on AI-generated specifications from Gemini.

Which port does the application use?

By default, the application runs on port 3000. You can change this by setting the PORT environment variable or using the -p flag with the Next.js dev command.

Usage Questions

What format should my specifications be in?

The application accepts two formats:

  • JSON: {"Color": "Red", "Material": "Steel"}
  • Colon-separated: Color: Red (one per line)

The parser automatically detects which format you're using.

How many attributes can I compare at once?

There's no hard limit, but we recommend 5-20 attributes for best performance. Too many attributes can slow down the AI response and make results harder to review.

What does "Show only differences" do?

This toggle filters the comparison table to show only attributes where your value differs from the AI-generated value. It helps you focus on potential issues without scrolling through matching attributes.

Why do some values show as mismatches when they seem the same?

The comparison is exact after normalization (lowercase, trimmed whitespace). Differences that cause mismatches include:

  • Spelling variations: "Aluminum" vs "Aluminium"
  • Unit differences: "10 lbs" vs "10 pounds"
  • Spacing: "256GB" vs "256 GB"
  • Specificity: "Blue" vs "Navy Blue"

Are case differences ignored?

Yes. The comparison is case-insensitive. "RED", "Red", and "red" are all considered matches.

Privacy & Security

Is my data sent to Google?

Only partially. When you run a comparison:

  • Sent to Google: The SKU and attribute keys (names only)
  • NOT sent: Your attribute values

This design protects your proprietary specification data while still enabling AI comparison.

Is my data stored anywhere?

No. The application doesn't use a database and doesn't store or log any comparison data. Each comparison is processed in real-time and not persisted.

Can others see my comparisons?

No. All comparisons happen client-side after receiving AI responses. There's no sharing or multi-user functionality.

Is this GDPR compliant?

The application itself doesn't collect or store personal data. However, if you're processing product data that contains personal information, you're responsible for ensuring compliance with relevant regulations.

Technical Questions

What AI model does this use?

The application uses Google's Gemini 2.5 Flash model, optimized for speed and accuracy.

How long does a comparison take?

Typically 2-5 seconds, depending on:

  • Number of attributes
  • Network latency
  • Gemini API load
  • Complexity of the SKU

Can I run this offline?

No. The application requires internet connectivity to access the Google Gemini API.

Does this work on mobile devices?

Yes. The application is fully responsive and works on smartphones and tablets. The interface adapts to smaller screens with horizontal scrolling for the comparison table.

What browsers are supported?

The application works on all modern browsers:

  • Chrome (recommended)
  • Firefox
  • Safari
  • Edge

Internet Explorer is not supported.

Can I customize the UI theme?

The application is designed for dark mode only. You can modify the Tailwind configuration in tailwind.config.ts if you need to change colors, but there's no built-in light mode.

Deployment Questions

Can I deploy this to production?

Yes. The application can be deployed to any platform that supports Node.js or Docker, including:

  • Vercel (recommended for Next.js)
  • Netlify
  • AWS (ECS, Lambda, Amplify)
  • Google Cloud (Cloud Run, App Engine)
  • Azure (App Service)
  • Self-hosted with Docker

How do I deploy with Docker?

See the Docker Deployment Guide for complete instructions. Quick start:

docker build -t sku-comparator .
docker run -p 3000:3000 --env-file .env.local sku-comparator

Does this scale horizontally?

Yes. The application is stateless and can run multiple instances behind a load balancer. Each instance only needs the GEMINI_API_KEY environment variable.

What are the system requirements?

Minimal requirements:

  • CPU: 1 core
  • Memory: 512MB minimum, 1GB recommended
  • Disk: ~200MB for application and dependencies
  • Network: Internet connectivity required

Troubleshooting

Why am I getting "GEMINI_API_KEY is not configured"?

This means the environment variable isn't being read. Check:

  • .env.local file exists
  • File is in project root directory
  • API key has no typos or extra spaces
  • Server was restarted after adding the file

Comparisons are failing with "Failed to generate comparison"

Common causes:

  • Invalid API key
  • Network connectivity issues
  • Gemini API quota exceeded
  • Gemini service temporarily unavailable

Check the Troubleshooting Guide for detailed solutions.

Input parsing keeps failing

Ensure your input matches one of the supported formats:

  • Valid JSON with double quotes
  • Or colon-separated lines with at least one colon per line

See Input Formats for examples.

The application won't start

Check:

  • Node.js version (18.17.0+)
  • Port 3000 isn't already in use
  • Dependencies are installed (npm install)
  • .env.local file exists with valid API key

Features & Limitations

Can I compare multiple SKUs at once?

No. Currently, the application supports one SKU comparison at a time. You need to run separate comparisons for each SKU.

Can I save comparison results?

No. The application doesn't include save/export functionality. Results are displayed only during the session and not persisted.

Can I customize the AI's behavior?

The AI model and system prompt are configured in app/api/compare/route.ts. You can modify the SYSTEM_PROMPT constant if you need different behavior, but this requires code changes.

Does it support languages other than English?

The UI is in English only. However, Gemini supports multiple languages for SKUs and specifications. The comparison will work with non-English attribute names and values.

Can I use a different AI model?

The application is designed for Google Gemini. Using a different model would require code changes in app/api/compare/route.ts to integrate with a different API.

Are there rate limits?

The application doesn't impose rate limits, but you're subject to Google Gemini's API rate limits based on your API key's quota. Monitor usage in Google AI Studio.

Best Practices

How can I get the most accurate comparisons?

  • Use specific, detailed SKUs
  • Include brand names when relevant
  • Use standard industry terminology
  • Be consistent with units and formatting
  • Focus on key differentiating attributes

When should I trust the AI vs my data?

Trust AI when:

  • SKU is well-known and widely available
  • Attributes use standard terminology
  • Product has been on market for a while

Trust your data when:

  • Product is custom or specialized
  • You have official manufacturer specifications
  • Using industry-specific terminology
  • Product is very new or unreleased

How often should I run comparisons?

Run comparisons:

  • When adding new products to your catalog
  • After bulk imports or data migrations
  • Periodically for quality assurance
  • When specifications seem inconsistent
  • Before publishing changes to production
