Quick Start Guide

Get up and running with your first SKU comparison in just a few minutes. This guide assumes you've already completed the installation process.

Your First Comparison

Let's perform a simple comparison to familiarize yourself with the application workflow.

Step 1: Start the Development Server

Open your terminal, navigate to the project directory, and start the server:

Terminal
npm run dev

Open http://localhost:3000 in your web browser.

Step 2: Enter a SKU

In the SKU input field, enter a product identifier. For this example, we'll use the default:

SKU-90210

Step 3: Input Specifications

The application accepts specifications in two formats. For this quick start, we'll use JSON format:

JSON Format
{
  "Color": "Crimson",
  "Material": "Aluminum",
  "Battery Life": "18 hours"
}

Copy this JSON and paste it into the "Your key-value pairs" textarea.

Step 4: Run the Comparison

Click the "Compare SKU" button. The application will:

  1. Parse your input specifications
  2. Send the attribute keys to Google Gemini
  3. Receive AI-generated values for those attributes
  4. Display a side-by-side comparison

You'll see a loading animation while the AI processes your request (typically 2-5 seconds).

Step 5: Review Results

The comparison table displays three columns:

  • Attribute: The specification field name
  • Your Value: The value you provided
  • LLM Value: The value generated by Google Gemini

Understanding Highlights

Rows where your value differs from the AI's value are highlighted with an amber background on the LLM Value cell. This makes it easy to spot discrepancies at a glance.

Trying Different Input Formats

Colon-Separated Format

You can also input specifications using a simple colon-separated format:

Colon-Separated Format
Color: Crimson
Material: Aluminum
Battery Life: 18 hours

The application automatically detects the format and parses it correctly.

Using the Filter Toggle

Above the comparison table, you'll find a "Show only differences" toggle switch.

  • Off (default): Shows all attributes in the comparison
  • On: Filters to show only attributes where values differ

This is particularly useful when comparing many attributes and you only want to focus on mismatches.

Understanding the Results

Match Summary

Below the input form, you'll see a summary showing:

  • Total number of attributes compared
  • Number of mismatches detected
Note: The comparison is case-insensitive and trims whitespace. "red" and "Red" are considered matches.

Interpreting Mismatches

When you see a mismatch, consider these possibilities:

  • Different terminology: You used "Aluminium" but the AI used "Aluminum"
  • Unit differences: You specified "18 hrs" but the AI used "18 hours"
  • More/less specific: You said "Blue" but the AI suggested "Navy Blue"
  • Actual difference: A genuine discrepancy that needs investigation

Common Scenarios

All Attributes Match

If all attributes match, you'll see:

All X attributes match Gemini's response.

This indicates strong alignment between your data and the AI's knowledge.

Some Attributes Differ

If some attributes differ, you'll see:

X mismatches detected across Y attributes.

Review the highlighted cells to identify differences.

No Results Yet

Before running your first comparison, you'll see:

Start a comparison to see Gemini's take on your SKU specifications.

Best Practices

  • Use descriptive SKUs: More specific SKUs help the AI provide better responses
  • Be consistent: Use the same terminology you use in your product catalog
  • Include relevant attributes: Focus on key specifications that define the product
  • Check units: Ensure consistent unit formatting (e.g., "GB" vs "gigabytes")
  • Review carefully: AI responses are estimates; verify critical information

What's Happening Behind the Scenes

When you click "Compare SKU," the application:

  1. Validates Your Input

    Checks that you've provided a SKU and at least one attribute

  2. Parses Specifications

    Converts your input (JSON or colon-separated) into a structured format

  3. Sends Request to API

    Transmits the SKU and attribute keys (not values) to the backend

  4. Queries Google Gemini

    The backend uses Gemini 2.5 Flash to generate specifications

  5. Enforces Structure

    Uses JSON schema to ensure the response matches your attribute keys exactly

  6. Displays Results

    Compares your values with AI values and highlights differences

Privacy Note

The application only shares your attribute keys with Google Gemini, not your values. Your proprietary specification data remains private while the AI generates its own interpretation based on the SKU and attribute names.
